# SupportAI — AI Support Platform

A multi-tenant AI support platform with RAG-powered chat, document ingestion, and automated ticket escalation.
Runs entirely on **free tiers** — no credit card required.

## Stack

| Layer | Service | Free tier |
|---|---|---|
| Frontend | Next.js 14 on **Vercel** | Unlimited hobby |
| Backend | Express + TypeScript on **Render** | 750 hrs/month |
| Database & Storage | **Supabase** | 500 MB DB, 1 GB storage |
| AI Chat | **Groq** (`llama-3.3-70b-versatile`) | 14,400 req/day |
| RAG / Search | PostgreSQL **full-text search** (tsvector) | Built into Supabase |

## Architecture

```
ai-support-platform/
├── frontend/     # Next.js 14, TypeScript, Tailwind, shadcn/ui
└── backend/      # Express API, TypeScript
    ├── src/
    │   ├── routes/       auth, documents, chat
    │   ├── services/     document, rag, escalation, ticket, embedding
    │   ├── middleware/   auth (JWT), error handler
    │   └── db/           supabase client
    └── supabase/
        └── schema.sql    full schema (run in Supabase SQL editor)
```

## RAG Pipeline

1. **Ingest** — upload PDF / DOCX / TXT / MD via `POST /api/documents/upload`
2. **Chunk** — text split into ~2000-char chunks with 200-char overlap
3. **Index** — each chunk stored in Postgres; `search_vector tsvector` column auto-generated by DB
4. **Retrieve** — `websearch_to_tsquery` FTS finds top-5 relevant chunks for each customer message
5. **Generate** — chunks injected into system prompt → Groq streams response via SSE

## Getting Started

### 1. Supabase

1. Create a project at [supabase.com](https://supabase.com)
2. Go to **SQL Editor** → paste contents of `backend/supabase/schema.sql` → Run
3. Go to **Storage** → create a bucket named `documents` (private)
4. Copy your **Project URL** and **anon key** from Project Settings → API

### 2. Groq

1. Sign up at [console.groq.com](https://console.groq.com) (free, no credit card)
2. Create an API key

### 3. Backend

```bash
cd backend
cp .env.example .env
# Fill in SUPABASE_URL, SUPABASE_KEY, GROQ_API_KEY, JWT_SECRET
npm install
npm run dev        # http://localhost:5000
```

### 4. Frontend

```bash
cd frontend
cp .env.local.example .env.local   # or edit .env.local
# Set NEXT_PUBLIC_API_URL=http://localhost:5000
npm install
npm run dev        # http://localhost:3000
```

| `SUPABASE_KEY` | Supabase anon key |
| `GROQ_API_KEY` | Groq API key |
| `JWT_SECRET` | Secret for signing JWTs (any long random string) |

### Frontend (`frontend/.env.local`)

| Variable | Description |
|---|---|
| `NEXT_PUBLIC_API_URL` | Backend base URL |

## API Routes

### Auth (all `/api/auth/`)
| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/register` | — | Create business + user + default ai_config |
| POST | `/login` | — | Returns JWT |
| POST | `/forgot-password` | — | Stub (returns 200) |
| GET | `/me` | JWT | Current user + business |

### Documents (all `/api/documents/`, JWT required)
| Method | Path | Description |
|---|---|---|
| POST | `/upload` | Upload file → chunk → index |
| GET | `/` | List business documents |
| DELETE | `/:id` | Delete document + chunks |
| POST | `/reindex` | Re-chunk all documents |

### Chat (all `/api/chat/`, public)
| Method | Path | Description |
|---|---|---|
| POST | `/` | SSE streaming chat with RAG |
| GET | `/widget-config/:businessId` | Bot name / welcome message |

## Deployment

### Render (backend)

1. New **Web Service** → connect repo → set root to `backend`
2. Build command: `npm install && npm run build`
3. Start command: `npm start`
4. Add all env vars from `.env.example`

### Vercel (frontend)

1. Import repo → set root to `frontend`
2. Add `NEXT_PUBLIC_API_URL` pointing to your Render URL

## SQL Migration (if upgrading from vector-based schema)

If you previously ran the pgvector schema, apply this migration:

```sql
-- Drop vector dependencies
ALTER TABLE chunks DROP COLUMN IF EXISTS embedding;
DROP INDEX IF EXISTS chunks_embedding_idx;

-- Add FTS column (Postgres auto-maintains it)
ALTER TABLE chunks
  ADD COLUMN IF NOT EXISTS search_vector tsvector
  GENERATED ALWAYS AS (to_tsvector('english', content)) STORED;

CREATE INDEX IF NOT EXISTS chunks_search_idx ON chunks USING GIN(search_vector);
CREATE INDEX IF NOT EXISTS chunks_business_idx ON chunks(business_id);
```
